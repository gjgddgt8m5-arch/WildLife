# WBS — Work Breakdown Structure

## Проєкт: WildLife 

Мобільний додаток, який об’єднує зоопарки та природні заповідники з різних країн і дозволяє користувачам у реальному часі спостерігати за життям тварин через онлайн-камери.

## 1. Planning
- Аналіз ідеї проєкту
- Визначення мети проєкту
- Визначення цільової аудиторії
- Визначення країн для запуску
- Формування команди
- Розподіл відповідальності між учасниками

## 2. Design
- Створення UX-структури додатку
- Створення UI-дизайну
- Підготовка прототипу
- Узгодження дизайну з командою

## 3. Backend
- Розробка серверної частини
- Створення бази даних
- Налаштування API
- Підключення онлайн-камер
- Налаштування безпеки даних

## 4. Mobile App
- Розробка мобільного додатку
- Створення основних екранів
- Інтеграція з backend
- Підключення відеопотоків
- Тестування роботи додатку на смартфонах

## 5. Testing
- Функціональне тестування
- Перевірка роботи камер
- Тестування реєстрації користувачів
- Виправлення помилок
- Повторне тестування після виправлень

## 6. Release
- Підготовка додатку до запуску
- Фінальна перевірка
- Публікація додатку
- Оголошення про запуск

## 7. Documentation
- Підготовка технічної документації
- Підготовка інструкцій для користувачів
- Опис структури проєкту
- Опис інтеграцій
